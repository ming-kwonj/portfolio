헤이홈 CX파트 / 전사 운영 적용
배경
상담 데이터가 비정형으로 쌓여 인사이트 도출이 불가능한 상태. 상담원마다 태깅 기준이 불분명해, 같은 케이스에 다른 태그를 붙이거나 유사한 태그 간 선택 혼선이 반복. AI 자동 분류를 적용하려면 일관된 기준이 필요.

Zapier + Gemini 태그 자동화 PoC
자체 기획, 개인 PoC 프로젝트
문제
Problem
VOC Taxonomy 설계 후, 상담원이 직접 태깅하는 구조에서 병목 발생을 확인함. 태그 기준을 명문화해도, 상담원 개인 성향에 따라 다르게 인지하거나, 특이건에 대해 태깅 방향을 문의하는 상황 발생. 상담이 쏟아지는 현장에서 매건 태그 검수는 비효율적인 리소스 운영이라고 판단해 "AI가 태그를 먼저 추천" 이라는 아이디어에서 시작.
시도
Approach
별도 예산 없이 개인 계정으로 Zapier 연결하여 동작 검증 시도. 구조는 단순하게 설계: 채널톡 상담 데이터를 Google Sheets로 내보낸 뒤, Zapier가 새 행을 감지하면 태그 정책을 학습한 Gemini API를 연동해 추천 태그를 다음 열에 자동 기입하는 플로우.

결과
Result
의도한 방향으로 동작 가능 확인. Gemini가 상담 내용을 읽고 T_/P_ 추천 태그를 기입. 실제 상담 데이터 검수 시, 높은 정확도로 구동하는 것 확인. "구조적으로 가능하다"는 것을 직접 검증한 것 자체에 의미를 둠.
한계
Limitation
채널톡 → Sheets 내보내기가 수동 배치라 실시간 상담 기반 동작이 아님. Zapier Free 플랜의 Task 및 Gemini API 호출 한도 및 보안성에 대한 우려점 확인. 채널톡 상담이 완료되는 시점에 자동으로 트리거되지 않으면 실 운영 적용이 어렵다는 구조적 한계 파악.
다음
Next Step
채널톡 Webhook을 외부 자동화 플로우와 직접 연결하는 방식으로 구조를 개선하면 실 운영 적용이 가능하다는 방향성 확립, 유료 플랜으로 전환 시 한도 및 보안 관련 우려 사항 개선 가능. PoC를 통해 자동화의 병목은 "데이터가 언제, 어떤 형태로 들어오는가"에 있으며, AI 프로세스에서는 모델 학습 방지 및 데이터 비식별화와 같은 "데이터 관리 원칙" 수립이 중요하다는 점 학습.
